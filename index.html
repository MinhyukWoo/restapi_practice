목표

Dentis DB에 대한 CRUD 기능을 HTTP의 JSON 방식으로 제공하는 RESTful API 서버 구현

진행사항

  • 모델 Organization에 대한 기초적인 CRUD 구현
  • 모델 Patient에 대한 기초적인 CRUD 구현
  • 모델 Treatment에 대한 기초적인 CRUD 구현

용어 설명

CRUD: Create, Read, Update, and Delete

소프트웨어의 기본적인 데이터 처리 기능이다.

  1. Create: 새로운 항목을 생성하는 기능이다.
  1. Read: 기존 항목을 읽는 기능이다.
  1. Update: 기존 항목을 갱신하는 기능이다.
  1. Delete: 기존 항목을 삭제하는 기능이다.

RESTful API

소프트웨어 아키텍처 REST를 바탕으로 한 웹 API이다.

REST: Representational State Transfer

개체 리소스를 주고 받는 것에 집중한다.

일반적으로 HTTP 프로토콜과 JSON 형식을 이용한다.

설계 원칙

  1. Uniform Interface: 균일한 표준 형식으로 정보를 전송한다.
  1. Statelessness: 이전의 모든 요청과 독립적으로 처리한다.
  1. Layered System: 클라이언트와 서버 사이의 다른 중간 서버가 처리할 수 있다.
  1. Cacheability: 중간 서버가 일부 응답을 저장하여 반환하는 캐시 기능을 지원한다.

이점

  1. 통신 병목 현상을 일으키지 않으면서 수많은 요청을 처리할 수 있다.
  1. 서버 애플리케이션의 변화가 클라이언트에 영향이 미치지 않는다.
  1. API 설계와 상관없이 다양한 프로그래밍 언어로 클라이언트 또는 서버 프로그래밍 할 수 있다.

HTTP: Hypertext Transfer Protocol

서버와 클라이언트가 통신하기 위한 프로토콜 중 하나이다.

클라이언트는 서버에게 HTTP Request를, 서버는 클라이언트에게 HTTP Response를 보낸다.

Stateless: 이전 HTTP Request의 상태를 저장하지 않는다.

Casing: 리소스 요청 시에 리소스의 복사본을 제공한다.

HTTP Method

HTTP Request에서 Method는 클라이언트가 요청하는 목적을 나타낸다.

  1. POST: 새로운 리소스를 등록하는 것을 요청한다. (Create에 대응)
  1. GET: 기존 리소스 정보를 읽는 것을 요청한다. (Read에 대응)
  1. PUT: 기존 리소스를 대체하는 것을 요청한다. (Update에 대응)
  1. DELETE: 기존 리소스를 삭제하는 것을 요청한다. (Delete에 대응)

JSON: JavaScript Object Notation

텍스트 기반의 데이터 교환 표준이다.

속도가 빠르고, 작성하고 사용하기 쉽다.

JSON Schema: 보낸 JSON 데이터가 유효한 것인지 확인할 수 있는 형식이다.

Serialization

모델의 객체를 HTTP로 전송할 수 있는 형식(JSON 등)으로 변환하는 것이다.

반대 과정을 Deserialization이라고 한다.

Serializer

Model의 instance와 Python built-in data type 간 변환을 담당한다.

데이터 유효성을 검증하는 is_valid, DB에 저장하는 save등이 있다.

위 기능을 제공하도록 직접 Serializer 클래스 정의하거나 ModelSerializer를 상속받는다.

Parser

HTTP Request에서 Python built-in data type으로 변환을 담당한다.

메서드parse(stream) 를 이용한다.

Render

Python built-in data type에서 HTTP Response로 변환을 담당한다.

메서드render(serializer.data) 를 이용한다.

class JsonResopnse(HttpResponse):
    def __init__(
        self, content: object = ..., *args: Any, **kwargs: Any
    ) -> None:
        content = JSONRenderer().render(content)
        kwargs['content_type'] = 'application/json'
        super().__init__(content, *args, **kwargs)


def __get_all(
    model_class: type[Model], serializer_class: type[Serializer]
):
    objs = model_class.objects.all()
    serializer = serializer_class(objs, many=True)
    return JsonResopnse(serializer.data)


def __insert(request: HttpRequest, serializer_class: type[Serializer]):
    parsed_data = JSONParser().parse(request)
    serializer = serializer_class(data=parsed_data)
    if serializer.is_valid():
        serializer.save()
        return JsonResopnse(
            serializer.data, status=status.HTTP_201_CREATED
        )
    else:
        return JsonResopnse(
            serializer.errors, status=status.HTTP_400_BAD_REQUEST
        )


def __delete_all(model_class: type[Model]):
    model_class.objects.all().delete()
    return HttpResponse(status=status.HTTP_204_NO_CONTENT)


def __db_overall(
    request: HttpRequest,
    model_class: type[Model],
    serializer_class: type[Serializer]
):
    if request.method == 'GET':
        return __get_all(model_class, serializer_class)
    elif request.method == 'POST':
        return __insert(request, serializer_class)
    elif request.method == 'DELETE':
        return __delete_all(model_class)
    else:
        return HttpResponse(status=status.HTTP_405_METHOD_NOT_ALLOWED)


def __db_detail(
    request: HttpRequest,
    pk: Any,
    model_class: type[Model],
    serializer_class: type[Serializer]
):
    try:
        obj = model_class.objects.get(pk=pk)
    except model_class.DoesNotExist:
        return HttpResponse(status=status.HTTP_404_NOT_FOUND)
    if request.method == 'GET':
        serializer = serializer_class(obj)
        return JsonResopnse(serializer.data)
    elif request.method == 'PUT':
        data = JSONParser().parse(request)
        serializer = serializer_class(obj, data=data)
        if serializer.is_valid():
            serializer.save()
            return JsonResopnse(serializer.data)
        else:
            return JsonResponse(
                serializer.errors, status=status.HTTP_400_BAD_REQUEST
            )
    elif request.method == 'DELETE':
        obj.delete()
        return HttpResponse(status=status.HTTP_204_NO_CONTENT)
    else:
        return HttpResponse(status=status.HTTP_405_METHOD_NOT_ALLOWED)


@csrf_exempt
def organization_overall(request: HttpRequest):
    return __db_overall(request, Organization, OrganizationSerializer)


@csrf_exempt
def organization_detail(request: HttpRequest, pk: int):
    return __db_detail(request, pk, Organization, OrganizationSerializer)


@csrf_exempt
def patient_overall(request: HttpRequest):
    return __db_overall(request, Patient, PatientSerializer)


@csrf_exempt
def patient_detail(request: HttpRequest, pk: int):
    return __db_detail(request, pk, Patient, PatientSerializer)


@csrf_exempt

    return __db_overall(request, Treatment, TreatmentSerializer)


@csrf_exempt
def treatment_detail(request: HttpRequest, pk: int):
    return __db_detail(request, pk, Treatment, TreatmentSerializer)

기초적인 CRUD를 수행하는 함수 __db_overall과 __db_detail을 구현

각각의 모델이 같은 CRUD를 수행할 수 있도록 매개변수로 해당 클래스를 받아 다형성 구현

클래스 JsonResponse를 직접 정의하여 HttpResponse에 JSON을 첨부하여 반환함

__db_overall(request, model_class, serializer_class)

GET: 해당 모델의 모든 항목을 전달하여 보냄

POST: 해당 모델에 새로운 항목을 추가한뒤 전달하여 보냄, 값이 올바르지 않으면 Bad Request

DELETE: 해당 모델의 모든 항목을 삭제한뒤 No Content를 보냄

이외 다른 메소드는 Method Not Allowed를 보냄

__db_detail(request, pk, model_class, serializer_class)

GET: 해당 모델의 pk와 일치하는 항목을 전달하여 보냄, 없으면 Not Found

PUT: 해당 모델의 pk와 같은 항목을 대체한 뒤 전달함, 값이 올바르지 않으면 Bad Request

DELETE: 해당 모델의 pk와 같은 항목을 삭제한 뒤 No Content를 보냄

이외 다른 메소드는 Method Not Allowed를 보냄

결론

정리된 사항

  1. RESTful API를 비롯한 필수적인 개념에 대해 이해했습니다.

    REST 아키텍처의 원칙 및 이점, HTTP 프로토콜, JSON 등 이해했습니다.

  1. Django와 Django Rest Framework의 전체적인 작업 흐름을 익혔습니다.

    Django의 views, urls 등과 Serializer, Parser, Renderer 등 사용하는 방법을 익혔습니다.

  1. 교재에 나오는 대로 dentis_db에 대해 기초적인 CRUD 기능을 구현했습니다.

    주어진 모델을 고치지 않으면서 RESTful API를 구현했습니다.

미해결된 사항

  1. 이미지 파일 POST하는 방법을 찾아야 합니다.

    모델 STL3D는 이미지 파일을 읽는데 JSON에 이미지 파일을 첨부하는 방법을 찾지 못했습니다.

  1. DELETE 기능에 대한 제한이 필요합니다.

    지금 제공하는 기능 중 일부는 데이터 전체를 삭제하는 등 클라이언트에 제공하기는 위험합니다.

    따라서 요구사항에 맞게 기능을 축소하거나 사용자 인증 과정이 필요합니다.

  1. 다른 HTTP method도 구현해야 합니다.

    Patch 방식은 Get과 다르게 데이터의 일부 필드 값만 전달해도 수정이 되게 요청하는 것입니다.

    따라서 기본적인 기능 외 다양한 Method에 대한 기능 구현 및 핸들이 필요합니다.
